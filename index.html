How to Apply This in Notion

Step-by-step setup guide — takes about 10 minutes

1
Create a New Page
In your Notion sidebar, click + New page. Title it 📊 Weekly WIP — Sales. Set the icon to 📊 and add a cover image (use a dark gradient or your brand photo). Press / to open the block menu at any time.
2
Add the Header Block
Type /callout → choose a colored background (e.g. dark gray). Inside it, write the week dates and team name. This acts as your banner. Above it, type /heading 1 for your page title.
3
Section 1 — Pros & Cons
Type /heading 2 → title: ✦ 01 — Week Highlights. Then type /2 columns to create a two-column layout. In the left column add a /callout with a green background for Pros. In the right column, add a /callout with red background for Cons. Use bullet lists inside each. Below that, add /3 columns for your 3 stat boxes using /callout blocks.
4
Section 2 — Sales Images
Type /heading 2 → title: 📸 02 — Sales of the Week. Then type /gallery to create a Gallery database. Each card = one sale. Add properties: Rep Name, Deal Value, Date Closed, Status (Won / Pending). Upload a screenshot or photo as the card cover image by opening a card and clicking Add cover.
5
Section 3 — Google Reviews
Type /heading 2 → title: ⭐ 03 — Google Reviews. Use a /database inline with columns: Reviewer, Stars (select: ★★★★★), Review Text, Date, Rep who requested it. Or simply use /callout blocks, one per review, with a star emoji prefix. Paste the review text inside each callout.
6
Style with Dark Theme & Colors
Go to Settings → Appearance → Dark mode to match this theme's dark aesthetic. Use block background colors: dark gray for neutral, green for wins, red for issues, yellow for pending. Set text color on headings to match sections. Pin the page to your sidebar and share it with your team via Share → Invite.
7
Duplicate as a Weekly Template
Once the page is set up, click the ••• menu → Duplicate at the start of each new week. Rename it with the new date range. Optionally, use /template button to automate this inside a parent WIP hub page.
Pro tip: Create a parent page called 📁 WIP Hub and nest each week's page inside it. Use a linked database view on the hub page to see all weekly reports at a glance. Add a Status property (Draft / Live / Archived) so the team knows which week is active.
